# מדריך התקנה — שיאים 2.0

מדריך צעד-אחר-צעד, בעברית פשוטה. לוקח בערך 20 דקות, עושים אותו **פעם אחת בלבד**.
בסוף: כולם נכנסים מקישור אחד, מכל מכשיר, בלי להדביק שום כתובת.

> **מי מבצע?** מומלץ שאהרון יבצע (מחשבון הגוגל של העסק), כי שם יושבים הנתונים הישנים ב-Drive.

---

## שלב 1 — פריסת השרת (Google Apps Script)

1. נכנסים אל [script.google.com](https://script.google.com) עם חשבון הגוגל של העסק.
2. לוחצים **New project** (פרויקט חדש).
3. מוחקים את הקוד שמופיע בעורך, ומדביקים במקומו את **כל** תוכן הקובץ `backend/Code.gs` מהריפו הזה.
4. שומרים (Ctrl+S). נותנים לפרויקט שם: `shiaim-server`.
5. לוחצים למעלה **Deploy ← New deployment**.
6. לוחצים על גלגל השיניים ⚙️ ובוחרים **Web app**.
7. ממלאים:
   - **Description:** `shiaim v2`
   - **Execute as:** `Me` (אני)
   - **Who has access:** `Anyone` (כל אחד) — אל דאגה: כל בקשה נבדקת עם סיסמה וטוקן בתוך הקוד.
8. לוחצים **Deploy**. גוגל יבקש אישור הרשאות — לוחצים **Authorize access**, בוחרים את החשבון, ואם מופיע מסך אזהרה לוחצים **Advanced ← Go to shiaim-server (unsafe)** ומאשרים. (זו אזהרה סטנדרטית על סקריפט פרטי שלא עבר בדיקת גוגל — זה הקוד שלנו, זה בסדר.)
9. מעתיקים את **Web app URL** (מתחיל ב-`https://script.google.com/macros/s/...`). שומרים בצד.

> הגיליון "שיאים — התנהלות שוטפת (נתונים)" והתיקייה "שיאים — התנהלות שוטפת" ב-Drive **נוצרים לבד** בכניסה הראשונה. לא צריך ליצור כלום ידנית.

## שלב 2 — הטמעת הכתובת בקוד

1. פותחים את הקובץ `js/config.js`.
2. מחליפים את `PASTE_APPS_SCRIPT_URL_HERE` בכתובת שהעתקתם:
   ```js
   const API_URL = 'https://script.google.com/macros/s/..../exec';
   ```
3. שומרים.

## שלב 3 — העלאה ל-GitHub Pages

1. יוצרים ריפו חדש ב-GitHub (למשל `shiaim-v2`) ומעלים אליו את **כל** תוכן התיקייה הזו (כולל תיקיית `backend` — כך קוד השרת שמור בריפו!).
2. בהגדרות הריפו: **Settings ← Pages ← Source: Deploy from a branch ← main ← / (root) ← Save**.
3. אחרי דקה-שתיים האתר חי בכתובת `https://<שם-המשתמש>.github.io/shiaim-v2/`.

## שלב 4 — הקמה ראשונית (סיסמאות)

1. פותחים את כתובת האתר בדפדפן.
2. יופיע מסך **"הקמה ראשונית"** — קובעים סיסמה ליעקב וסיסמה לאהרון. (המסך הזה מופיע פעם אחת בלבד; הסיסמאות נשמרות מגובבות בשרת ותקפות מכל מכשיר.)
3. מתחברים: שם משתמש `yakov` או `aharon` + הסיסמה שנקבעה.

## שלב 5 — ייבוא הנתונים מהמערכת הישנה

1. נכנסים כ-`yakov` (רק המנהל יכול לייבא).
2. **הגדרות ← ייבוא מהמערכת הישנה ← התחל ייבוא**.
3. מדביקים את כתובת ה-Apps Script של המערכת **הישנה** (נמצאת אצל אהרון) ולוחצים.
4. המערכת מייבאת: פרויקטים, עיצובים, הערות, סטטוסים, יומן שינויים — והופכת אוטומטית כל שם לקוח לכרטיס לקוח. שום דבר לא נמחק במערכת הישנה, ובטוח להריץ שוב אם משהו נתקע (אין כפילויות).
5. בודקים שהכל נמצא, ואז אפשר להפסיק להשתמש במערכת הישנה.

## שלב 6 — התקנה על הטלפון (PWA)

- **אנדרואיד (Chrome):** פותחים את האתר ← תפריט ⋮ ← **הוספה למסך הבית**.
- **אייפון (Safari):** פותחים את האתר ← כפתור שיתוף ← **הוסף למסך הבית**.

## עדכון גרסה בעתיד

1. משנים את הקוד בריפו (או מבקשים מ-Claude Code…).
2. מעלים גרסה במשתנה `CACHE_VERSION` בקובץ `sw.js` (למשל `shiaim-v2.0.1`).
3. דוחפים ל-GitHub — המשתמשים מקבלים את העדכון אוטומטית בטעינה הבאה.

אם שיניתם משהו ב-`backend/Code.gs`: מדביקים את הקוד המעודכן ב-script.google.com ← **Deploy ← Manage deployments ← ✏️ ← Version: New version ← Deploy**. הכתובת נשארת אותה כתובת — לא צריך לגעת ב-config.js.
